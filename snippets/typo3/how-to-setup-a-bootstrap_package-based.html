How to setup a bootstrap_package based site package in TYPO3

by View all posts by Sigala Mbigha, sigalambigha.home.blog ▪ Montag, 13. April 2020, 2:40

https://www.sitepackagebuilder.com/

A site packages is a TYPO3 extension used to package and hold together the files and configurations needed to customize our TYPO3 installation.

This is the recommended method of implementing websites in TYPO3.

A site package based on the bootstrap_package extension is the best to use as it comes with many additional features and configurations. These makes it very easy for you to get started implementing your website.

But if you follow the steps to setup a site package from this article for a bootstrap_package based site package created from sitepackage builder, you will get the errors below.

  1. Error below when trying to open a page in the frontend after setting up the sitepackage.

2. The flash message error below when trying to create a new page in the Page module

This article will show you another way of setting up a bootstrap_package based sitepackage and avoid having the errors above.

Note: test_package in the error above is the name of my sitepackage extension’s folder.

You are already facing the errors above??

If you are already facing the errors above, follow the steps here to fix each error and continue with setting up the sitepackage

ERROR 1: ScssPhp\ScssPhp\Exception\CompilerException

  1. Delete the sitepackage symlink folder found in the public/typo3conf/ext folder. Where public is the document root folder.
  2. Run **composer remove mycompany/test-package** in the terminal to remove your site package from composer. mycompany/test-package is the package name and can be gotten from the sitepackage’s composer.json file.
  3. Extract the downloaded sitepackage downloaded earlier from the Sitepackage Builder into the public/typo3conf/ext folder.
  4. Continue with the “Register site package’s PHP class names in Composer“ section below.

ERROR 2: Page tree error

This error occurs because bootstrap_package which has already been installed needs some additional database fields which have not yet been created. bootstrap_package was automatically installed because the Extension Manager was set to automatically install extensions after downloading them from the TER.

You can solve this error by following the “Update database“ steps below or just deactivating and activating again the bootstrap_package extension using the Extension Manager.

Setting up a new sitepackage and have not faced the errors above ?

If you want to setup a new sitepackage and did not follow the steps in this article, then start from here..

Creating the site package

  • Go to sitepackagebuilder.com and click on “CREATE SITEPACKAGE” button.

  • On the next page, fill in the form. Below are screenshots showing an example of how I did mine.

  • Click on “Create Sitepackage” button at the end of the form.

  • On the next page, check the “Generated Configuration” information. If everything is correct, click on the “DOWNLOAD” button to download your site package.

  • Extract the downloaded .zip sitepackage file to the **public/typo3conf/ext** folder of your TYPO3 project folder.

  • Note: “public” is the document root. This folder may have a different name in your project. Go here to see how to determine this.

Register site package’s PHP class names in Composer

Add an autoload property to the project’s composer.json file (That is the myProjectFolder/composer.json file). The autoload property should look like below.

"autoload": {
    "psr-4": {

    }
  }

To this autoload property, add your sitepackage’s namespace mapping to paths. Get this information from the site package’s composer.json file (the composer.json file in the root of your sitepackage folder). See sample screenshot below for more info.

Sitepackage’s composer.json file

So, a part of the project’s final composer.json file including the modified autoload property will look something like below.

Part of project’s composer.json file

  • Next, run composer req bk2k/bootstrap-package command. To download the bootstrap_package extension. Take note to download the version compatible with your TYPO3 version.
  • Go to the ADMIN TOOLS > Extensions module (Extension Manager) and activate the bootstrap_package and sitepackage extensions if they have not yet been activated.

You have to update the project’s database. If not you will get “A Page tree error” when creating pages in the page tree. This is going to affect some operations in the page tree. Follow steps below to update the database.

  1. Go to “ADMIN TOOLS —> Maintenance” submodule.
  2. Click on “Analyze database” button.
  3. Select all the database changes to be made under the sections “Add tables” and “Add fields to tables”.
  4. Scroll to the bottom of the modal and click on the “Apply selected changes” button.
  5. After database updates made, close “Analyze Database Structure” modal.

Next is to setup the site package so it works with your TYPO3 installation. So, lets roll ……

Setting up the sitepackage

Next is to create a root page which is going to hold our TypoScript template. Follow the steps below to create the page.

  • Go to the “WEB —> Page” sub-module and create a new Page under the Page tree item having the TYPO3 logo.
  • Enable the page and click on the just created page.
  • Click on the Edit page button then click on the Behaviour tab.
  • Turn on the “Use as Root Page” option.
  • Click on Save button to save the changes.

Next is to create a TypoScript template for our website. To do that, follow the steps below:

  • Click on the Root page you just created to select it.
  • Go to the “WEB —> Template” submodule and click the “Create template for a new site” button.
  • Switch from “Constant Editor” to “Info/Modify” in document header (Found at top of the page).
  • Click “Edit the whole template record” button.
  • Remove auto-generated TypoScript in the “Setup” section of the page.
  • Go to the “Includes” tab and in the part to “Include static (from extensions)“, select the sitepackage’s static template(it has the same name as the extension).
  • Click on Save button to save the change.
  • Click on the “View” button to view your the created page.

You should see a page that looks like below.

This shows you have successfully configured your site package. You can now make the necessary modifications or configurations to implement your website.

Conclusion

So there you go. You have successfully configured a bootstrap_package based site package. If any questions or issues, just leave them in the comments and lets discuss.
